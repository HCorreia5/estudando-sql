# Dia 00 — Introdução
- Objetivo Principal do SQL: Tomar melhores decisões consultando, organizando e armazenando dados.
- SQL é a linguagem universal para interagir com bancos de dados relacionais.
- Sua principal função é consultar (pedir), manipular (alterar) e definir (criar) dados.

## Por que SQL é Fundamental?
- Permite acesso direto aos dados, sem intermediários.
- É a base para tomar decisões de negócio informadas.
- É um pré-requisito para praticamente todas as carreiras em Dados (Análise, Ciência, Engenharia).

## Os 3 Pilares da Área de Dados:
- Engenharia de Dados: Coleta, armazena e disponibiliza os dados.
- Análise de Dados: Explica o que aconteceu e por quê (passado/presente).
- Ciência de Dados: Prevê o que vai acontecer (futuro).

## Categorias de Comandos SQL:
- `DQL ( Query )` : Para fazer perguntas ( `SELECT` ). É o foco principal do curso.

---

- `DML ( Manipulation )` : Para gerenciar os registros ( `INSERT`, `UPDATE`, `DELETE` ).

---

- `DDL ( Definition )` : Para gerenciar a estrutura das tabelas ( `CREATE`, `DROP` ).

## Estrutura do Curso (Visão Geral):
- DQL (Data Query Language): `SELECT`, `WHERE` - Para fazer perguntas/consultas.
- Agregações: `GROUP BY`, `COUNT()` - Para resumir e agrupar dados.
- JOINs: `INNER`, `LEFT` - Para combinar dados de múltiplas tabelas.
- DDL (Data Definition Language): `CREATE`, `DROP` - Para criar/deletar tabelas.
- DML (Data Manipulation Language): `INSERT`, `UPDATE` - Para adicionar/modificar dados.

## A Consulta Fundamental:
SELECT colunas FROM tabela; → O esqueleto de toda consulta de dados.
